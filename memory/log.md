## 2026-07-01 — ИДЕЯ ДЖОНА: вариант «кольцо Сатурна» = опорное кольцо-противовес (НЕ низ-балансир)
- Случайный хром-обод, который я сделал на UFO-модели, Джон хочет как **отдельный концепт-вариант крепления/качания.** Идея: кресло держится и качается **на КОЛЬЦЕ (как кольцо Сатурна), а не на нижнем куполе-утяжелителе** (как у UFO). Кольцо = опора + противовес, кресло балансирует в нём/на нём.

## 2026-07-01 — Нижнее 3D пересобрано из НАСТОЯЩЕГО меша USDZ
- **Нижнее кресло = скан магазинного кресла-референса FONTE FT-30003** (этикетка **98×100×142 см**, ротанг PE / OLEFIN подушка, до 100 кг, ~5980 бат). НЕ пенопластовое кресло Джона. Это форма-референс для 3-го кресла.
- Старый рендер (облако красных точек `THREE.Points`) выглядел «не сделано». Тело вращения из профиля `chair_clean` дало «горшок» (кресло асимметричное — вращением не передать хвост+открытое сиденье).
- **Решение (по подсказке CONTEXT.md):** взял реальный меш из `data/scan-kreslo.usdz` (Apple ObjectCapture, 12496 верш / 25000 полигонов, up=Y, метры). usd-core (`python3 -m pip install --user usd-core`). Скрипт: applyLocalToWorldTransform → ×1000 мм → центр X/Z, пол Y=0 → экспорт `data/chair_mesh.json` (v+i, 609 КБ).
- Габариты меша ~**100×104×139 см** = совпали с этикеткой → меш уже стоит вертикально (Y-up), PCA-ось НЕ нужна для постановки (только для профиля).
- В `index.html` нижний блок (THREE2/canvas c2): `fetch('data/chair_mesh.json')` → BufferGeometry(pos+index) → computeVertexNormals → MeshStandardMaterial ротанг 0xbf9d6e DoubleSide. Пол уменьшен (1100/2200), 2 directional света, cr2=2.6, midY=650. Крутится мышкой, не качается. Console чистая, network 200.
- **Следующее:** 3-е кресло = форма как FONTE (верх) + низ-неваляшка (качание по кругу как верхнее). + проблема «верхнее слишком низкое».
